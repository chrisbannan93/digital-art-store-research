## 2) File Naming & Packaging Rules (deterministic)

### A. ZIP naming standard (final form)
- Single: `{{SKU_CODE}}_printable-kit_v{{VERSION}}.zip`
- Trio: `{{SKU_CODE}}_trio-printable-kit_v{{VERSION}}.zip`
- Example version token: `v1`, `v2`, `v3`.

### B. Internal artwork naming
- Base pattern: `{{SKU_CODE}}_ART-{{LETTER}}_{{RATIO_CODE}}_v{{VERSION}}.jpg`
- Allowed `{{LETTER}}`: `A` (Single), `A|B|C` (Trio)
- Allowed `{{RATIO_CODE}}`: `2x3`, `3x4`, `4x5`, `11x14`, `ISO-A`

### C. Ratio pack naming
- Folder naming pattern: `ratio_{{RATIO_CODE}}`
- Example folders: `ratio_2x3`, `ratio_3x4`, `ratio_4x5`, `ratio_11x14`, `ratio_ISO-A`

### D. Versioning rules
- Use whole-number release versions only (`v1`, `v2`, `v3`).
- Increase version when any customer-facing file content changes.
- Keep one active latest ZIP in product delivery; archive previous ZIPs in `/archive/`.
- Do not mix file versions inside a ZIP (all files must match ZIP version).
