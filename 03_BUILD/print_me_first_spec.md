# PRINT_ME_FIRST.pdf Blueprint Spec

## 1) Page Order (exact)
1. Cover + quick reassurance
2. What you received in this order
3. Choose your print path
4. Ratio-to-frame match guide
5. Test print checklist
6. Common fixes (non-technical)
7. License + support reminders

---

## 2) Page Intents (no final prose)

### Page 1 — Cover + quick reassurance
- Intent: reduce anxiety immediately.
- Must include:
  - Product identifier (`{{SKU_CODE}}`)
  - Core process line: `Download -> Print -> Frame`
  - Canonical digital line: `Digital download only. No physical item is shipped.`

### Page 2 — What you received in this order
- Intent: confirm expected files and avoid confusion.
- Must include:
  - ZIP summary
  - Ratio summary (5 ratios)
  - Single vs Trio count summary
  - `PRINT_ME_FIRST.pdf` purpose line

### Page 3 — Choose your print path
- Intent: offer simple route choices.
- Must include:
  - Home printer path
  - Local print shop path
  - Online print lab path
  - Calm guidance to choose easiest option

### Page 4 — Ratio-to-frame match guide
- Intent: remove frame-size uncertainty.
- Must include all ratio families:
  - 2:3
  - 3:4
  - 4:5
  - 11:14
  - ISO (A-series)
- Must include plain instruction to match ratio before printing.

### Page 5 — Test print checklist
- Intent: reduce waste and first-attempt errors.
- Must include:
  - Test print first reminder
  - Scale check
  - Paper choice check
  - Color expectation check
  - Frame size re-check

### Page 6 — Common fixes (non-technical)
- Intent: solve predictable issues without jargon.
- Must include issue categories:
  - Wrong crop/fit
  - Color looks different
  - File won’t open
  - Size mismatch
- Must include support escalation line.

### Page 7 — License + support reminders
- Intent: set boundaries and support path confidence.
- Must include:
  - `Personal Use Only` reminder
  - Not allowed reminders (resale/redistribution/file sharing)
  - Support-first flow: troubleshoot -> resolve -> refund if unresolved within policy window
  - Support contact placeholder: `{{SUPPORT_CONTACT}}`
  - Support SLA placeholder: `{{SUPPORT_SLA}}`

---

## 3) Language Rules
- Calm, parent-safe, non-hype tone.
- Short sentences and plain words.
- Avoid technical printing jargon unless required, and define it in simple terms.
- Keep instruction style actionable and reassuring.

## 4) Must-Never-Promise Rules
- Do not promise exact color matching across all printers.
- Do not promise instant support replies outside stated SLA.
- Do not promise commercial-use rights.
- Do not promise physical delivery.
- Do not promise refunds outside support-first policy conditions.

## 5) License + Support Placement Rules
- License reminder appears on page 7 (required).
- Support path reminder appears on page 6 and page 7 (required).
- Canonical digital-only line appears on page 1 (required).
